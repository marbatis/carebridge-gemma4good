# CareBridge Video Script

Target:
- 2 minutes 30 seconds to 3 minutes
- built for YouTube upload
- optimized for the Gemma 4 Good Hackathon judging criteria

## Core Message

CareBridge helps caregivers make safer escalation decisions in rural, low-connectivity, and postpartum scenarios by combining grounded retrieval with Gemma 4 narrative generation.

## Opening Hook

Suggested voiceover:

`A mother gives birth, goes home, and ten days later develops a severe headache, chest pain, and swelling. In a rural county, that can turn into a life-threatening delay before anyone realizes this is an emergency. CareBridge is built for that moment.`

Visuals:
- title card
- one strong screenshot of the CareBridge interface
- short on-screen text:
  - `Local-first`
  - `Grounded`
  - `Powered by Gemma 4`

## Problem

Suggested voiceover:

`Many health tools are too generic, too internet-dependent, or too vague when a caregiver needs clear next steps. In rural and low-connectivity settings, missing a red flag can be dangerous.`

Visuals:
- simple map or rural-access visual
- form fields showing weak connectivity, transport limits, postpartum context

## Product Demo

Suggested voiceover:

`Here is CareBridge in action. A caregiver enters a postpartum case with severe headache, chest pain, and swelling. The system classifies the case, retrieves grounded safety notes, and uses Gemma 4 to generate a caregiver-facing response.`

Visuals:
- screen recording:
  - user enters postpartum scenario
  - submits form
  - response appears
- pause on:
  - `urgency: emergency`
  - warning signs
  - handoff card

Suggested line while paused on response:

`The response is not a generic chatbot answer. It gives urgency, warning signs, immediate steps, and a handoff card a family can actually use with a nurse line, clinic, or emergency team.`

## Why Gemma 4

Suggested voiceover:

`Gemma 4 is used for the narrative layer. The deterministic system controls risk level, retrieval grounding, and escalation structure. Gemma 4 turns that into a clearer, more human-readable explanation that works better for caregivers under stress.`

Visuals:
- simple architecture slide
- boxes:
  - `Caregiver Input`
  - `Safety Backbone`
  - `Grounded Retrieval`
  - `Gemma 4 Narrative Layer`
  - `Handoff Card`

## Technical Credibility

Suggested voiceover:

`This prototype runs on Kaggle using a real Gemma 4 runtime path. To make it robust, I added grounded retrieval, deterministic backstops, runtime validation, and fallback logic for partial model outputs.`

Visuals:
- Kaggle notebook status showing successful completion
- small snippet from successful JSON output
- code or architecture diagram

## Closing Impact

Suggested voiceover:

`CareBridge shows how Gemma 4 can do more than answer questions. It can help families make better decisions in moments when systems are overloaded, far away, or hard to reach. This is a narrow prototype today, but the same pattern can expand to maternal care, pediatric triage, multilingual support, and climate-health resilience.`

Visuals:
- return to product UI
- closing slide:
  - `CareBridge`
  - `Health & Sciences`
  - `Gemma 4 Good Hackathon`

## Production Notes

- Keep the first 20 seconds strong.
- Show the live app by 30 seconds.
- Do not spend too long on setup or architecture before the demo.
- Add subtitles.
- Use clean zooms on the handoff card and urgency output.
- End with one memorable sentence, not a feature list.
